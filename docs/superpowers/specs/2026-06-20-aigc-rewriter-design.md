# 中文论文降 AIGC 率 Skill 设计文档

## 概述

设计一个 Claude Code skill，用于降低中文学术论文的 AIGC 检测率。核心思路：**Python 做检测仪，Claude 做手术刀，模式库做参考手册**。

目标平台：知网（主要）、维普、万方、PaperPass 等国内 AIGC 检测平台。

目标学科：建筑/工程方向（主要），兼顾通用学术。

第一版支持格式：纯文本（`.txt`）和 Markdown（`.md`）。Word（`.docx`）和 LaTeX（`.tex`）作为后续迭代。

## 运行模式

### 交互模式（主模式）

用户逐段粘贴文本，skill 分析后输出改写结果。

```
用户：/rewrite
skill：当前处于交互模式，请粘贴需要处理的段落。
用户：（粘贴一段文字）
skill：[分析 → 改写 → 输出结果 + 风险分变化]
用户：（继续粘贴下一段）
...
用户：/done
skill：本次处理了 N 段，平均风险分从 X 降到 Y。
```

**交互模式状态管理：**

| 用户指令 | 行为 |
|---------|------|
| 粘贴新段落 | 正常处理，追加到本次会话记录 |
| `重来` / `再来一次` | 对上一段重新改写（保留原文，换策略） |
| `太激进了` / `保守点` | 当前段切换为轻度改写 |
| `再大胆些` | 当前段切换为深度改写 |
| `换风格 xxx` | 切换 `--style` 参数（有效值：`academic` / `narrative` / `technical`），后续段落生效。输入无效值时提示可用选项 |
| `/done` | 结束会话，输出本次处理汇总 |

会话内维护一个段落列表，记录每段的原文、改写结果、风险分变化。`重来` 操作替换列表中最后一段的结果。重复粘贴相同段落视为"重来"。

### 半自动模式

读取文件，标记高风险段落，用户选择性处理。

```
用户：/rewrite --file paper.md --mode semi
skill：[读取 → 分析 → 输出风险报告]
      高风险段落：第3段(0.82)、第7段(0.75)、第12段(0.71)
      是否逐段处理？
用户：处理第3段和第7段
skill：[逐段改写 → 输出结果]
```

### 全自动模式

读取文件，全文处理，diff 预览，确认后写入新文件。

```
用户：/rewrite --file paper.md --mode auto
skill：[预检 → 分析 → 全文改写 → 准确性检查]
      处理完成，风险分：0.78 → 0.25
      发现 3 处可疑改写需要确认：
      1. "围护结构" → "外围护结构"（建议保留原文）
      ...
      请确认后将写入 paper_rewritten.md
用户：确认
skill：已写入 paper_rewritten.md，diff 报告已保存至 paper_diff.md
```

## Python 分析引擎

### 分析维度

| 维度 | 检测内容 | 风险信号 |
|------|---------|---------|
| 句法特征 | 句长方差、从句嵌套深度、并列结构频率 | 句长过于均匀、从句嵌套过深、大量并列 |
| 词汇分布 | TTR、连接词频率、套话检测 | "综上所述""值得注意的是"等 AI 高频表达 |
| 结构规律 | 段落长度方差、段首句模式、论证节奏 | 段落等长、每段首句结构相同、论证节奏机械 |
| AI痕迹 | 困惑度估算、突发性 | 过于流畅、缺乏口语化断句、无个人化表达 |
| 中文特化 | "了"字集中度、"的"字冗余嵌套、四字成语密度、"被…所…"频率 | 中文 AIGC 检测的特有信号 |

### 优先级排序

输出按 `风险分 × 段落重要性` 排序。段落重要性由章节类型决定：

- 讨论 > 方法 > 摘要 > 相关工作 > 结论 > 引言 > 未匹配章节（默认权重 1.0）

"未匹配章节"指无法通过标题关键词识别的正文段落，使用权重 1.0（即优先级等于风险分本身）。

### 输出格式

```json
{
  "overall_risk": 0.72,
  "paragraphs": [
    {
      "index": 0,
      "risk": 0.85,
      "priority": 0.77,
      "section_type": "discussion",
      "issues": [
        {"type": "cliche", "detail": "检测到套话: '综上所述'"},
        {"type": "uniform_length", "detail": "连续3段句长方差<5"},
        {"type": "chinese_de", "detail": "'的'字嵌套超过3层"}
      ],
      "suggestion": "重点改写句式结构，替换连接词"
    }
  ]
}
```

### 技术选型

- 纯 Python，无重型依赖
- 中文分词：jieba
- 句法分析：规则 + 统计，不依赖大模型
- 可独立运行：`python analyze.py input.txt --output report.json`

## 模式库

### 三层结构

**第一层：内置规则（`patterns/builtin.json`）**

随 skill 分发，用户不应修改：
- 200+ 条 AI 高频表达及替换建议
- 50+ 条句式变换模板
- 各检测平台的已知特征权重

**第二层：用户规则（`patterns/user.json`）**

用户可自由编辑：
- 术语保护名单（如"建筑能耗模拟""围护结构"不可替换）
- 个人写作偏好
- 针对特定平台的自定义策略

**第三层：自动积累（`patterns/learned.json`）**

运行时自动发现并追加，**每次运行结束写盘，下次运行加载**：
- 改写后风险分仍高于阈值的表达 → 标记为"顽固 pattern"
- 用户手动拒绝的替换建议 → 记录为"不适用"
- 每次运行后更新命中率统计
- 支持 `--no-learn` 参数防止测试运行污染规则库

### 规则文件格式

```json
{
  "patterns": [
    {
      "id": "cliche_001",
      "type": "cliche",
      "match": "综上所述",
      "replacements": ["从整体来看", "回到开头的问题", "总结前文"],
      "platform_weight": {"cnki": 0.9, "vip": 0.7},
      "source": "builtin",
      "hits": 0,
      "success_rate": 0.0,
      "examples": {
        "before": "综上所述，本文提出了一种基于深度学习的方法。",
        "after": "回到开头的问题，本文提出的方案根植于深度学习。"
      }
    }
  ],
  "protected_terms": ["建筑能耗模拟", "围护结构", "DeST"]
}
```

## 改写执行层

### 风险分级策略

| 风险分 | 策略 | 具体操作 |
|--------|------|---------|
| 0.3–0.5 | 轻度改写 | 替换连接词、调整语序、打破并列结构 |
| 0.5–0.7 | 中度改写 | 长短句拆合、主被动互换、插入过渡句、增加口语化断句 |
| 0.7+ | 深度改写 | 段落重组、增加主观标记、引入非典型论证节奏 |

### 改写风格（`--style` 参数）

| 风格 | 说明 | 适用场景 |
|------|------|---------|
| `academic`（默认） | 正式学术语气 | 通用 |
| `narrative` | 叙述性风格 | 建筑学交叉方向 |
| `technical` | 紧凑技术风格 | CS/AI 方向 |

### 改写约束（prompt 硬规则）

1. **术语保护**：用户规则中的 `protected_terms` 和论文关键词表中的术语原样保留
2. **含义保真**：语义不得偏离原文，允许表述方式变化，不允许内容增删
3. **学术语气**：不变成口语，只是从"AI 标准体"变成"真人学术体"
4. **公式/表格/引用**：一律跳过，只处理正文文字
5. **每段独立迭代**：各段独立判断是否达到阈值，达标即停止该段

### 上下文窗口管理

改写某段时，采用滑动窗口策略：
- "段"指自然段落（以空行分隔的文本块），非句子
- 携带前 2 个自然段 + 后 2 个自然段作为上下文
- 如果当前段位于文档开头/末尾，不足 2 段时有多少带多少
- 保持语义连贯的同时控制 token 量
- 分析阶段全文扫描，改写阶段分窗口执行

### 准确性验证（改写后自动执行）

逐句对比原文与改写结果，标记三类可疑项：
- 术语被替换（如"围护结构"变成了"外围结构"）
- 关键数据/数值有变化
- 逻辑关系改变（因果变并列、条件变让步）

可疑项高亮呈现给用户确认。全自动模式下所有可疑项列出后暂停等待确认。

## 章节感知的差异化阈值

### 章节识别方式

通过标题关键词匹配自动识别章节类型。"标题"的判定：
- Markdown 文件：以 `#` 开头的行
- 纯文本文件：独占一行且长度 < 30 字的短行（视为隐式标题）

匹配规则（不区分大小写，支持中英文）：

- **摘要**：`摘要`、`abstract`、`提要`
- **引言**：`引言`、`绪论`、`introduction`、`问题提出`
- **方法**：`方法`、`方法论`、`实验`、`method`、`methodology`、`实验设计`、`模型构建`
- **结果**：`结果`、`实验结果`、`results`、`数据分析`
- **讨论**：`讨论`、`分析与讨论`、`discussion`、`结果分析`
- **结论**：`结论`、`总结`、`conclusion`、`结语`
- **相关工作**：`相关工作`、`文献综述`、`related work`、`研究现状`

无法匹配的段落默认归为"正文"，使用标准阈值 0.3。

### 各章节默认阈值

| 章节 | 默认阈值 | 理由 |
|------|---------|------|
| 摘要 | 0.25 | 高度可见，检测重点 |
| 引言 | 0.3 | 标准 |
| 方法 | 0.35 | 技术描述天然偏公式化 |
| 结果 | 0.3 | 数据描述需保持准确 |
| 讨论 | 0.25 | 最能体现个人风格，改写收益最大 |
| 结论 | 0.3 | 标准 |
| 相关工作 | 0.4 | 综述体天然偏高，放宽 |

用户可通过 `--threshold` 参数全局覆盖。设置后所有章节统一使用该值，不再按章节差异化。

## 输出文件规范

### 全自动模式生成文件

- `<原文件名>_rewritten.md`：改写后的完整文档
- `<原文件名>_diff.md`：逐段对比报告（表格格式，见下）
- `<原文件名>_analysis.json`：完整分析报告（供复查）

### diff 报告格式

使用 Markdown 表格，每段一行：

```markdown
| 段落 | 章节 | 原文风险 | 改写风险 | 原文 | 改写结果 | 可疑项 |
|------|------|---------|---------|------|---------|--------|
| §3 | 方法 | 0.82 | 0.28 | 本文提出了一种基于...的方法，该方法具有... | 本文的方案根植于...，其优势在于... | 无 |
| §7 | 讨论 | 0.75 | 0.31 | 综上所述，实验结果表明... | 回到实验本身，数据指向... | "实验结果"→"数据"（建议确认） |
```

段落列显示段落序号，原文和改写结果列截取前 50 字并加 `...` 后缀。完整文本见 `_rewritten.md`。

### 迭代控制

- 默认阈值：风险分 ≤ 0.3 停止（按章节差异化）
- 最多迭代 3 轮，3 轮后仍高于阈值则提示用户手动介入
- 每轮改写后输出当前分数，让用户看到进度

### Token 成本参考

全自动模式处理一篇约 10000 字的论文（~50 段）：
- 单轮改写：约 15-20 万 input token + 输出
- 含迭代（2-3 轮）：约 30-60 万 input token
- 交互模式和半自动模式按实际处理段落计，通常远低于全自动

## 调用方式

| 调用方式 | 效果 |
|---------|------|
| `/rewrite` | 进入交互模式，等待用户粘贴段落 |
| `/rewrite --file paper.md` | 读取文件，进入半自动模式 |
| `/rewrite --file paper.md --mode auto` | 读取文件，进入全自动模式 |
| `/rewrite --file paper.md --mode semi` | 读取文件，进入半自动模式（显式指定） |
| `/done` | 交互模式下结束当前会话 |

`--file` 存在时默认进入半自动模式；显式指定 `--mode` 可覆盖。

## 参数总览

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--file` | 输入文件路径（省略则进入交互模式） | 无 |
| `--mode` | 运行模式：`interactive` / `semi` / `auto`（需配合 `--file` 使用） | `interactive` |
| `--style` | 改写风格：`academic` / `narrative` / `technical` | `academic` |
| `--threshold` | 风险分阈值 | 0.3（按章节差异化） |
| `--no-learn` | 不将本轮发现写入 learned.json | false |

## 错误处理

| 异常场景 | 检测方式 | 应对策略 |
|---------|---------|---------|
| 文件不存在或路径错误 | 读取前检查 `os.path.exists` | 提示用户检查路径，不继续执行 |
| 不支持的文件格式 | 扩展名检查 | 提示"第一版仅支持 .txt 和 .md"，建议用户转换格式 |
| 文件为空或无正文内容 | 读取后检查段落数 | 提示"未检测到可处理的文本内容" |
| 用户粘贴空内容 | 交互模式下检查 | 提示"请粘贴需要处理的段落" |
| 单段过长（>2000字） | 字数检查 | 按句号/分号拆分为多个子段，每个子段独立参与后续流程（独立分析、独立改写、独立迭代）。滑动窗口中，子段之间的上下文取相邻子段 |
| 改写后风险分不降反升 | 改写前后对比 | 自动回退到原文，尝试换策略重写一次；仍失败则标记为"处理困难段落"并跳过 |
| 分析引擎执行失败 | Python 进程退出码 | 降级为纯 prompt 改写（无量化指标），提示用户"分析引擎不可用，将使用基础改写模式" |
| 模式库文件损坏/格式错误 | JSON 解析异常 | 跳过损坏的规则文件，加载其余层，提示用户修复 |
| 迭代 3 轮仍未达阈值 | 迭代计数 | 停止该段迭代，标记为"需人工处理"，继续其他段落 |

**通用原则**：单段处理失败不阻塞全文流程。错误信息以 `[警告]` 前缀输出，最终汇总中列出所有未成功处理的段落。

**交互模式补充：**

| 异常场景 | 应对策略 |
|---------|---------|
| 粘贴内容无明显句号/分号（无法合理分句） | 提示"该段落缺少标点，建议先手动分句再处理" |
| 用户指令无法识别 | 提示可用指令列表 |
| 用户在无活跃段落时输入"重来" | 提示"当前没有可重写的段落" |
