# 中文边界情况处理详解

## 短文本（少于50字）

### 问题
太短的文本改写空间有限，改写后可能反而质量下降。

### 识别特征
- 少于50个汉字
- 只有一个简单句
- 缺乏上下文

### 处理方式
1. 告诉用户："这段文本太短，改写效果有限。建议提供更长的段落。"
2. 如果用户坚持，仍然尝试改写，但说明效果可能不理想。

### 示例
```
用户：帮我改写 "该方法有效。"
回复：这段文本太短（5字），改写空间有限。建议提供至少一个完整句子（50字以上）。是否仍然需要改写？
```

---

## 非学术文本

### 问题
技能专为学术论文设计，非学术文本可能效果不佳。

### 识别特征
- 口语化表达：我觉得、我认为、其实、反正
- 网络用语：666、yyds、绝绝子
- 日常用语：你好、谢谢、再见
- 商业文案：欢迎选购、限时优惠

### 处理方式
1. 识别非学术文本
2. 告诉用户："这个技能专为学术论文设计，非学术文本可能效果不佳。"
3. 如果用户坚持，仍然尝试改写，但说明可能不适用。

### 示例
```
用户：帮我改写 "这个方法真的太好用了！"
回复：这段文本不是学术文本，这个技能专为学术论文设计。是否仍然需要改写？
```

---

## 已经低相似度的文本

### 问题
如果文本本身相似度就低，改写可能反而破坏质量。

### 处理方式
1. 先运行相似度计算
2. 如果相似度已经很低（连续13字匹配 < 5%），告诉用户："这段文本相似度已经很低，不需要改写。"
3. 如果用户坚持，仍然尝试改写。

### 示例
```
用户：帮我改写这段文本
回复：[运行相似度计算] 这段文本的相似度已经很低了，不需要改写。是否仍然需要改写？
```

---

## 纯英文文本

### 问题
技能专为中文论文设计，纯英文文本可能效果不佳。

### 处理方式
1. 识别纯英文文本
2. 告诉用户："这段文本是英文，建议使用英文降重技能。"
3. 如果用户坚持，仍然尝试改写。

### 示例
```
用户：帮我改写 "The model was used to calculate the water yield."
回复：这段文本是英文，建议使用英文降重技能（paper-rewriter）。是否仍然需要改写？
```

---

## 中英混合文本

### 问题
中英混合文本可能影响改写效果。

### 识别特征
- 包含英文单词或句子
- 专业术语用英文标注
- 中英文交替出现

### 处理方式
1. 识别英文部分，保留不改
2. 只改写中文部分
3. 告知用户："检测到中英混合文本，只改写中文部分。"

### 示例
```
原文：本研究采用InVEST模型对生态系统服务进行评估。
改写：本文运用InVEST模型对生态系统服务进行分析。
（保留"InVEST"不改）
```

---

## 大量公式/引用的文本

### 问题
公式和引用不能改，改写空间有限。

### 识别特征
- 包含数学公式
- 包含 [1], [2] 引用
- 公式/引用占比超过50%

### 处理方式
1. 识别公式和引用，标记为"不改"
2. 只改写可改写的部分
3. 告知用户："文本中包含大量公式/引用，改写空间有限。"

### 示例
```
原文：根据公式 $E = mc^2$ [1]，可以计算出...
改写：依据公式 $E = mc^2$ [1]，能够计算出...
（公式和引用不改，只改写其他部分）
```

---

## 超长文本（超过1000字）

### 问题
一次性改写太长的文本可能质量下降。

### 处理方式
1. 建议用户分段提供
2. 如果用户坚持，分段改写，每段不超过500字
3. 告知用户："文本较长，建议分段改写以保证质量。"

### 示例
```
用户：帮我改写这2000字的段落
回复：文本较长（2000字），建议分段改写以保证质量。我可以先改写前500字，然后继续处理剩余部分。是否这样处理？
```

---

## 列表/枚举文本

### 问题
列表结构不适合改写。

### 识别特征
- 以 - 或 • 开头
- 编号列表 1. 2. 3.
- 项目符号列表

### 处理方式
1. 保留列表结构
2. 只改写列表项的内容
3. 告知用户："检测到列表结构，保留结构，只改写内容。"

### 示例
```
原文：
1. 第一个因素是温度。
2. 第二个因素是降水。
3. 第三个因素是土壤类型。

改写：
1. 首要因素为温度。
2. 次要因素为降水。
3. 第三因素为土壤类型。
```

## 表格/图片标题

### 问题
标题通常很短，改写空间有限。

### 识别特征
- 表1、图1、Table 1、Fig. 1
- 标题文字通常少于20字

### 处理方式
1. 告诉用户："表格/图片标题通常很短，改写效果有限。"
2. 如果用户坚持，尝试改写，但说明效果可能不理想。

### 示例
```
用户：帮我改写 "图1 研究区位置图"
回复：表格/图片标题通常很短，改写效果有限。是否仍然需要改写？
```

---

## 直接引语

### 问题
直接引语是引用他人原话，不能改。

### 识别特征
- 用引号括起来的部分
- "..." 或 '...'
- 直接引用他人的话

### 处理方式
1. 识别直接引语（用引号括起来的部分）
2. 保留直接引语不改
3. 只改写其他部分

### 示例
```
原文：张三(2020)指出："该方法具有重要价值。"
改写：张三(2020)认为："该方法具有重要价值。"
（直接引语不改，只改写转述部分）
```

---

## 术语密集文本

### 问题
术语太多，改写空间有限。

### 识别特征
- 包含大量专业术语
- 术语占比超过30%
- 难以找到同义词

### 处理方式
1. 识别所有术语，标记为"不改"
2. 只改写非术语部分
3. 告知用户："文本中术语密集，改写空间有限。"

### 示例
```
原文：InVEST模型采用habitat quality模块和carbon storage模块进行生态系统服务评估。
改写：InVEST模型运用habitat quality模块和carbon storage模块开展生态系统服务评估。
（术语不改，只改写连接词和动词）
```

---

## 参考文献列表

### 问题
参考文献有固定格式，不能改。

### 识别特征
- [1] 作者. 标题. 期刊, 年份, 卷(期): 页码.
- 参考文献列表

### 处理方式
1. 识别参考文献列表
2. 告诉用户："参考文献有固定格式，不能改写。"
3. 如果用户坚持，说明改写可能破坏格式。

---

## 代码片段

### 问题
代码有特定语法，不能改。

### 识别特征
- 代码块
- 变量名、函数名
- 编程语言语法

### 处理方式
1. 识别代码片段
2. 保留代码不改
3. 只改写注释和文档字符串

---

## 数学方程

### 问题
数学方程有固定格式，不能改。

### 识别特征
- 数学符号 +, -, *, /, =, <, >
- 希腊字母 α, β, γ, Δ, Σ
- 数学函数 sin, cos, log

### 处理方式
1. 识别数学方程
2. 保留方程不改
3. 只改写方程周围的文字说明

---

## 重复表达

### 问题
同一概念在文中多次出现，需要保持一致性。

### 处理方式
1. 识别重复出现的概念
2. 使用不同的同义词替换
3. 保持全文一致性

### 示例
```
原文：
- 研究表明X很重要。
- 研究表明Y很重要。
- 研究表明Z很重要。

改写：
- 研究表明X具有重要意义。
- 研究显示Y发挥关键作用。
- 研究揭示Z是核心因素。
```

---

## 处理原则

1. **主动告知**：遇到边界情况，主动告诉用户
2. **保留原意**：无论如何改写，不能改变原意
3. **保留格式**：公式、引用、代码等格式不能破坏
4. **用户优先**：如果用户坚持，仍然尝试改写
